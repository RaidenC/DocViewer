DocViewer - Tree Behavior

How should the file tree handle expansion and loading?

A

Lazy Load on Expand

Click folder → API call to load children. Shows spinner while loading. Caches loaded children. Good for large datasets.

B

Eager Load Initial Levels

Load first 2-3 levels upfront. Subsequent levels load on expand. Faster initial experience for typical use.

C

Search-First Mode

Default view shows flat list of recent documents. Tree only appears when clicking "Browse" or typing in search.
